# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in this project, please **do not** disclose it publicly before it has been reviewed.

Instead, please report it by creating a **private security advisory** (if available) or by contacting the maintainers through one of the following methods:

- Open a GitHub Security Advisory (preferred), if enabled.
- If Security Advisories are unavailable, please create a GitHub Issue describing the vulnerability **without including sensitive exploit details**, and we will provide instructions for secure communication if needed.

We will acknowledge reports as soon as reasonably possible and work to investigate and resolve confirmed issues.

---

## Abuse Reports and Responsible Use

If you have concerns about potential abuse, misuse, or policy compliance related to this repository, please **create an Issue** on this repository with the relevant details.

Please include:
- A description of your concern.
- Relevant links or evidence.
- Steps to reproduce the issue, if applicable.

We review all legitimate inquiries and will take appropriate action when necessary.

---

## Scope

This policy applies only to this repository and its officially maintained source code.
